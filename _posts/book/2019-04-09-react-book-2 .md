### JSX란?

JSX는 자바스크립트의 확장 문법이며 XML과 매우 비슷하게 생겼습니다.이런 형식으로 작성한 코드는 브라우저에서 실행되기 전에 코드가 번들링되는 과정에서 바벨을 사용하여 일반 자바스크립트 형태의 코드로 변환됩니다.

```javascript
// JSX문법
function App() {
  return (
    <div>
      Hello <b>react</b>
    </div>
  );
}

// 변환된 코드
function App() {
  return React.createElement(
    'div',
    null,
    'Hello ',
    React.createElement('b', null, 'react')
  );
}
```

**JSX 장점 : 보기 쉽고 익숙하다. (가독성이 좋다.)**

### JSX 문법

**1. 감싸인 요소**

- 컴포넌트에 여러 요소가 있다면 부모요소 하나로 감싸야 한다.

```javascript
  // src/App.js
  import React from 'react';

  //에러 발생
  function App() {
    return (
      <h1>리액트 안녕!</h1>
      <h2>잘 작동하니?</h2>
    );
  }

  export default App;

  //정상 코드 (div를 Fragment ,<>로 변경가능)
  function App() {
    return (
      <div>
        <h1>리액트 안녕!</h1>
        <h2>잘 작동하니?</h2>
      </div>
    );
  }

  export default App;
```

**2. 자바스크립트 표현**

```javascript
import React from 'react';

function App() {
  const name = '리액트';
  return (
    <div>
      <h1>{name} 안녕!</h1>
      <h2>잘 작동하니?</h2>
    </div>
  );
}

export default App;
```

**3. if문 대신 조건부 연산자**

```javascript
import React from 'react';

function App() {
  const name = '리액트';
  return (
    <div>
      {name === '리액트' ? <h1>리액트입니다.</h1> : <h2>리액트가 아닙니다.</h2>}
    </div>
  );
}

export default App;
```

**4. AND 연산자(&&)를 사용한 조건부 렌더링**

```javascript
import React from 'react';

function App() {
  const name = '뤼액트';
  return <div>{name === '리액트' ? <h1>리액트입니다.</h1> : null}</div>;
}

export default App;

// &&연산자를 사용하여 더 간단하게 표현가능
function App() {
  const name = '뤼액트';
  return <div>{name === '리액트' && <h1>리액트입니다.</h1>}</div>;
}
```

- && 연산자로 조건부 렌더링을 할수 있는 이유는 리액트에서 false를 렌더링할 때는 null과 마찬가지로 아무것도 나타나지 않기 때문입니다.

**5. undefined를 렌더링하지 않기**

```javascript
// src/App.js
import React from 'react';
import './App.css';

//오류발생
function App() {
  const name = undefined;
  return name;
}

export default App;

//정상코드
function App() {
  const name = undefined;
  return name || '값이 undefined입니다.';
}

//name 값이 undefined일때 보여 주고 싶은 문구가 있다면 다음과 같이 코드 작성가능
function App() {
  const name = undefined;
  return <div>{name || '리액트'}</div>;
}
```

**6. 인라인 스타일링**

    background-color처럼 -문자가 퐇람되는 이름이 아닌 backgroundColor로 작성해야 한다.(카멜표기법)

**7. class 대신 className**

    HTML에서 CSS 클래스 사용시에는 class ="example"이 아닌 className="example"을 사용해야한다.

**8. 꼭 닫아야 하는 태크**

    태그를 꼭닫아한다 , self-closing 태그를 통해 선언하면서 동시에 닫는다.

**9. 주석**

```javascript
// src/App.js
import React from 'react';
import './App.css';

//오류발생
function App() {
  const name = '리액트';
  return (
    <>
      {/* 주석은 이렇게 작성합니다. */}
      <div
        className="react" // 시작 태그를 여러 줄로 작성하게 된다면 여기에 주석을 작성할수 있음
      >
        {name}
      </div>
      // 하지만 이런 주석이나 /* 이런 주석은 페이지에 그대로 나타나게 됩니다. */
      <input />
    </>
  );
}

export default App;
```
